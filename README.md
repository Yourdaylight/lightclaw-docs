# LightClaw Docs

LightClaw 官方文档站，基于 [Docusaurus 3](https://docusaurus.io/) 构建。

线上地址：**https://lightclaw.clawtown.cn**

---

## 本地开发

```bash
# 安装依赖（只需执行一次）
npm install

# 启动本地预览（热重载，改文件立即生效）
npm start
```

浏览器打开 http://localhost:3000 即可预览。

## 内容维护

### 文档目录结构

```
docs/
├── getting-started/          # 快速开始（固定顺序）
│   ├── intro.md
│   ├── installation.md
│   └── quickstart.md
├── user-guide/
│   ├── core/                 # 核心功能
│   └── ui/                   # 使用界面
├── config/
│   ├── basic/                # 基础配置
│   └── advanced/             # 高级配置
├── arch/
│   ├── core/                 # 核心架构
│   └── data-security/        # 数据与安全
└── dev/
    ├── setup/                # 环境与项目
    └── contribute/           # 开发与贡献
```

### 新增文档（自动识别，无需修改配置）

在对应子目录下新建 `.md` 文件即可，侧边栏会**自动识别并显示**：

```bash
# 示例：新增「Telegram 渠道」文档
vim docs/config/basic/channels-telegram.md
```

文件头部加上标题：

```markdown
---
title: Telegram 渠道接入
sidebar_position: 3   # 可选，控制该文档在分组内的排序
---

正文内容...
```

> `sidebar_position` 越小越靠前，不写则按文件名字母序排列。

### 新增分类目录

如需在已有分组下增加子分类：

1. 新建目录，例如 `docs/user-guide/core/tutorials/`
2. 在 `sidebars.ts` 对应位置加一行：

```ts
{ type: 'autogenerated', dirName: 'user-guide/core/tutorials' },
```

3. 往新目录放 `.md` 文件，自动出现。

### 修改首页

首页在 `src/pages/index.tsx`，CSS 在 `src/pages/index.module.css`。

## 部署到线上

### 一键部署脚本

```bash
./deploy.sh
```

该脚本会自动完成：`npm run build` → `rsync 上传到服务器` → `nginx reload`

> 服务器：`43.132.178.232`，文件目录：`/var/www/lightclaw-docs/`

### 手动部署（分步）

```bash
# 1. 构建
npm run build

# 2. 上传（需要 sshpass）
sshpass -p 'lightclaw@1234' rsync -az --delete \
  -e "ssh -o StrictHostKeyChecking=no" \
  build/ root@43.132.178.232:/var/www/lightclaw-docs/

# 3. Reload nginx
sshpass -p 'lightclaw@1234' ssh -o StrictHostKeyChecking=no root@43.132.178.232 \
  "kill -HUP \$(ps aux | grep 'nginx: master' | grep -v grep | awk '{print \$2}' | head -1)"
```

### 完整更新流程

```bash
# 1. 编辑或新增文档
vim docs/config/basic/channels-telegram.md

# 2. 本地预览确认
npm start

# 3. 一键上线
./deploy.sh

# 4. 推送到 git（可选，保留历史）
git add .
git commit -m "docs: 新增 Telegram 渠道文档"
git push
```

## SSL 证书

当前使用临时证书，待正式证书后执行：

```bash
# 方式一：腾讯云申请免费证书后上传
scp lightclaw.clawtown.cn_bundle.crt root@43.132.178.232:/etc/nginx/ssl/

# 更新 /etc/nginx/conf.d/lightclaw-docs.conf 中的证书路径，然后 reload

# 方式二：开放安全组 80 端口后，自动签发
sshpass -p 'lightclaw@1234' ssh root@43.132.178.232 \
  "certbot --nginx -d lightclaw.clawtown.cn --non-interactive --agree-tos -m your@email.com --redirect"
```

## nginx 配置

配置文件在服务器 `/etc/nginx/conf.d/lightclaw-docs.conf`，支持：

- SPA 路由（`try_files` fallback）
- 静态资源长缓存（js/css/图片 1 年）
- HTTP → HTTPS 重定向
